Building Basecamp Project Stacks with Hotwire

Originally published on the 37signals dev blog · 2023-11-07

This post covers how we shipped Stacks — a way to visually group projects in Basecamp — using Hotwire and Rails. What made it interesting was how little new code it actually took, largely because of how well the existing architecture supported the change.

Stacks lets Basecamp users collapse related projects into a named group on the home screen. Under the hood, the feature uses Rails delegated types to model stacks as a lightweight layer over existing project records. The drag-and-drop behavior, already present in the app, required only small additions to handle the new grouping logic. Turbo and Stimulus handled the dynamic UI updates without reaching for a heavier JavaScript solution.

The post walks through the reasoning behind these choices and how keeping the model and controller changes focused made the whole feature easier to reason about and maintain.

I wrote this during my time at 37signals. It reflects the kind of problems we enjoyed solving there — real features, real constraints, real Rails.
